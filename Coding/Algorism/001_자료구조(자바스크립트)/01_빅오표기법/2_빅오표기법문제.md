# 1. 개념 
왜 n² + n + 자연수 → O(n²)가 되냐?
- 데이터가 커질수록, n²이 나머지 항(n, 자연수)을 완전히 압도하기 때문이야.
> 풀이하는 법 
1. n의 지수가 가장 큰 것을 찾기 
2. n 앞에 붙은 숫자를 삭제하기 
3. O()로 감싸기

# 3. 오답문제풀이 
> 14번 
```
`for i in range(2 * n): pass`  
`for j in range(3 * n): pass`
```
- 2n + 3n = 5n → O(n)
- for문이 분리되어 있다. 

> 19번
```
`for i in range(n): pass`  
`for j in range(n*n): pass`
```
- n + n² → 지배항 n²
- O(n^2)

> 24번
```
3n² + 2n² + n
```
- 3 + 2 = 5n², n은 무시

>27번
```
n log n + 10n
```
- O(n log n)
- n log n > n, 지배항 n log n

> 30번
```
5n² log n + n² + n
```
-  로그 
	- “어떤 수를 몇 번 곱해야 목표 숫자가 나오느냐?”
	예를 들어:
	- 2x=82^x = 82x=8  
	    → 2를 몇 번 곱해야 8이 될까? 2 × 2 × 2 = 8  
	    → x = 3 → 이게 로그: log⁡28=3\log_2 8 = 3log2​8=3
	- 10x=100010^x = 100010x=1000  
	    → 10 × 10 × 10 = 1000  
	    → x = 3 → log⁡101000=3\log_{10} 1000 = 3log10​1000=3
- O(n² log n)
- n² log n > n² > n, 지배항 n² log n
